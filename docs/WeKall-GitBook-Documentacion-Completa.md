# Configuración WebPhone

La sección **Configuración** permite al usuario personalizar y revisar los ajustes principales de su cuenta, audio, producto Wekall y opciones adicionales del Webphone.<br>

En este bloque se muestran los datos principales del usuario conectado.

* **Organización:** nombre de la empresa o grupo al que pertenece el usuario.
* **Correo electrónico:** dirección asociada a la cuenta del usuario.
* **Dirección SIP:** identificador utilizado para conectarse al sistema PBX o Contact Center.
* **Rol:** nivel de permisos dentro del sistema (por ejemplo: `USER`, `ADMIN`, etc.).

<figure><img src="https://content.gitbook.com/content/eN7PEbma2qR1S3UsZOiw/blobs/9RuJqbjJWoXEhJWwraEI/image.png" alt=""><figcaption></figcaption></figure>

#### Producto Wekall

Permite seleccionar el producto de la plataforma con el cual el webphone se conectará.

* **Seleccionar producto:** desplegable donde se elige entre los distintos servicios disponibles, como por ejemplo:
  * `PBX`
  * `Contact Center`
  * Otros productos disponibles según la organización.

#### Audio

Aquí se configuran los **dispositivos de entrada y salida de audio** para las llamadas.

**Micrófono:** Permite seleccionar el dispositivo de entrada de audio (micrófono) que se usará en las llamadas.

**Altavoz – Audio en la llamada:** Selecciona el dispositivo por donde se escuchará la voz del interlocutor durante la llamada.

**Altavoz – Notificación llamada entrante:** Define el altavoz que reproducirá el tono de llamada cuando entre una llamada.

**Volumen de llamada entrante:** Control deslizante para ajustar el nivel de volumen del timbre de llamadas entrantes.<br>

#### Autorespuesta

Configura la función de **auto contestar llamadas**.

* **Autorespuesta:** permite habilitar o deshabilitar esta función.
  * `Habilitada` → el sistema responde automáticamente las llamadas según el tiempo configurado.
* **Tiempo de autorespuesta:** define cuántos segundos deben pasar antes de que se conteste la llamada automáticamente (campo deshabilitado si la configuración está administrada por la organización).

{% hint style="info" %}
Esta configuración puede ser administrada
{% endhint %}

#### Zoom

Controla el **nivel de zoom de la interfaz** del webphone.

* Botones **“–”** y **“+”**: reducen o aumentan el tamaño de la interfaz.
* Valor central muestra el nivel actual (por ejemplo: **100%**).

#### Controles generales

* **Botón de cerrar**: ubicado en la esquina superior derecha, permite salir de la pantalla de configuración y regresar a la aplicación principal.

Actualmente, el WebPhone soporta las siguientes integraciones:

## Conceptos Clave

### Registro Manual vs. Registro Automático

Es importante entender la diferencia entre estos dos modos de registro:

#### Registro de llamadas

Cuando el registro está habilitado, al finalizar una llamada el agente verá un **formulario** donde podrá completar los campos configurados (título, nota, campos personalizados) antes de enviar el registro al CRM. El agente decide qué información incluir y cuándo enviar el formulario.

#### Registro automático activado

Cuando el registro automático está activado, el sistema crea **automáticamente** un objeto en el CRM **en el momento en que inicia la llamada**, sin intervención del agente. El tipo de objeto creado depende del CRM:

* **HubSpot**: Se crea un **Engagement** (registro de actividad de llamada)
* **Salesforce**: Se crea un **Task** (tarea asociada a la llamada)
* **Freshdesk**: Se crea un **Ticket** (ticket de soporte)

Al crearse automáticamente, el sistema muestra inmediatamente la **URL del objeto creado** en el CRM, permitiendo al agente acceder directamente al registro mientras la llamada está en curso.

> **Importante:** El registro automático solo funciona cuando está explícitamente activado en la configuración. Si no está activo, el registro será siempre manual mediante formulario al finalizar la llamada.

## Secciones de Configuración

Cada integración se configura desde el **Panel de Administración** del WebPhone y cuenta con tres secciones principales:

**Modalidad / Credenciales**: Define cómo y dónde opera la integración **Configuración de Campos**: Define qué información se registra con cada llamada **Registro de Llamadas**: Controla qué tipos de llamadas se registran y si el registro es manual o automático

Consulte la documentación específica de cada integración para obtener instrucciones detalladas de configuración.


# Freshsales

La integración con Freshsales permite registrar y sincronizar la información de llamadas realizadas desde el Webphone de WeKall hacia tu cuenta de Freshsales mediante el uso de **API REST autenticada**.

Esta integración requiere configuración manual de credenciales y endpoint, permitiendo una comunicación directa entre el Webphone y Freshsales.

<figure><img src="https://2310468671-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FeN7PEbma2qR1S3UsZOiw%2Fuploads%2FfYiu2NR0EbtwRHDlZ3C8%2Fimage.png?alt=media&#x26;token=cdbe85b0-245f-408e-b6a6-847cf7e28547" alt=""><figcaption></figcaption></figure>

### Configuración

La integración se compone de los siguientes parámetros:

#### 1. Estado

Permite habilitar o deshabilitar la integración.

* **Activado**: el Webphone enviará información de llamadas a Freshsales.
* **Desactivado**: no se enviará información y los agentes no podrán registrar llamadas en Freshsales.

#### Access Token

Token de autenticación proporcionado por Freshsales.

* Se utiliza para autorizar las solicitudes hacia la API de Freshsales.
* Debe configurarse correctamente para permitir la creación y actualización de registros.

#### URL

Endpoint de la API de Freshsales donde se enviará la información.

* Corresponde a la URL base o específica del servicio de Freshsales.
* Es utilizada por el Webphone para enviar los eventos de llamadas.

### Funcionamiento

#### Flujo general:

1. El agente realiza o recibe una llamada desde el Webphone
2. El Webphone gestiona el ciclo de vida de la llamada
3. Al ocurrir eventos relevantes (inicio, fin, tipificación, etc.), se realiza una llamada HTTP a la API de Freshsales
4. Freshsales procesa la información y:
   * Registra la llamada
   * La asocia a un contacto o lead existente
   * Puede crear registros si está configurado para ello


# Zoho CRM

La integración con Zoho CRM permite a los agentes gestionar llamadas directamente desde el entorno de Zoho, asegurando que toda la actividad quede registrada dentro del CRM.

Esta modalidad funciona de forma **embebida dentro de Zoho CRM**, centralizando la operación del agente en una sola herramienta.

<figure><img src="https://2310468671-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FeN7PEbma2qR1S3UsZOiw%2Fuploads%2FKUNc8enlvvyeuAcj3NFX%2Fimage.png?alt=media&#x26;token=bb8dfdfb-06db-4893-9417-cb122657da2f" alt=""><figcaption></figcaption></figure>

La integración cuenta con una configuración basada en la modalidad de uso:

#### 1. Modalidad de Zoho

Define cómo los agentes utilizarán la integración:

* **Únicamente dentro de Zoho**:\
  Los agentes deberán operar exclusivamente desde Zoho CRM para gestionar llamadas.

### Restricción de uso

Cuando la modalidad **“Únicamente dentro de Zoho”** está activa:

* Los agentes **solo podrán usar la integración desde Zoho CRM**
* No podrán utilizar el Webphone de forma independiente
* Todas las llamadas (entrantes y salientes) deberán gestionarse dentro del CRM

### Funcionamiento

#### Flujo general:

1. El agente accede a Zoho CRM
2. Desde Zoho, utiliza la funcionalidad de llamadas integrada
3. El Webphone opera de forma embebida o integrada en este entorno
4. Las llamadas:
   * Se realizan o reciben desde Zoho
   * Se registran automáticamente en los módulos correspondientes (contactos, leads, etc.)


# Clientify

La integración con Clientify permite sincronizar la información de las llamadas realizadas desde el Webphone de WeKall hacia tu cuenta de Clientify mediante un mecanismo basado en **webhooks**.

Esta integración es **desacoplada** y no requiere la instalación de componentes embebidos ni plugins dentro de Clientify.

<figure><img src="https://2310468671-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FeN7PEbma2qR1S3UsZOiw%2Fuploads%2FGSb4DeacDlJKhUf7gICw%2Fimage.png?alt=media&#x26;token=c806a02d-168a-4ba5-bf14-50da644d3ebd" alt=""><figcaption></figcaption></figure>

La configuración de la integración es simple y consta de dos parámetros:

#### 1. Estado de la integración

Permite habilitar o deshabilitar el envío de eventos hacia Clientify.

* **Activado**: el Webphone enviará información de llamadas al webhook configurado.
* **Desactivado**: no se enviará ningún evento a Clientify.

#### 2. URL de Clientify (Webhook)

Campo donde se debe ingresar la URL proporcionada por Clientify para la recepción de eventos.

* Esta URL corresponde a un **endpoint HTTP** que recibirá la información de las llamadas.
* Es responsabilidad de Clientify procesar esta información para:
  * Crear o actualizar contactos
  * Registrar llamadas en el timeline
  * Asociar la actividad al cliente correspondiente


# HubSpot

La integración con HubSpot permite registrar llamadas entrantes, salientes y transferidas directamente como **engagements** (registros de actividad) en el CRM de HubSpot. Esta integración funciona tanto desde el modo PBX (Business Phone) como desde el Contact Center.

<figure><img src="https://2310468671-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FeN7PEbma2qR1S3UsZOiw%2Fuploads%2Fma3lMASKrPUkcDaObyFa%2Fimage.png?alt=media&#x26;token=bdd3b1c2-48c8-40ef-99c6-3387c9b37d40" alt=""><figcaption></figcaption></figure>

## 1. Modalidad de HubSpot

La modalidad controla desde dónde los agentes pueden utilizar la integración con HubSpot para registrar llamadas.

Para configurar la modalidad, acceda a **Panel de Administración → Integraciones → HubSpot → Modalidad de HubSpot** y seleccione una de las siguientes opciones:

| Modalidad                        | Descripción                                                                                                                                                                                                                              |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Dentro y fuera de HubSpot**    | Los agentes pueden registrar llamadas tanto cuando están trabajando dentro de la interfaz de HubSpot como cuando utilizan el WebPhone de forma independiente. Esta es la opción más flexible y recomendada para la mayoría de los casos. |
| **Únicamente dentro de HubSpot** | Los agentes solo pueden registrar llamadas cuando están trabajando dentro de la interfaz web de HubSpot. Si el agente está utilizando el WebPhone fuera de HubSpot, no podrá registrar la llamada.                                       |

> **Recomendación:** Si sus agentes trabajan tanto dentro como fuera de HubSpot durante su jornada, seleccione "Dentro y fuera de HubSpot" para garantizar que todas las llamadas puedan ser registradas.

## 2. Configuración de Campos

Esta sección define los campos que el agente debe completar al registrar una llamada en HubSpot. Los campos aparecerán en el formulario de registro que se muestra al finalizar la llamada (registro manual).

### Campos Predeterminados

Estos campos vienen configurados por defecto y **no pueden ser eliminados**, solo editados:

| Campo      | Tipo  | Descripción                                                                                                             |
| ---------- | ----- | ----------------------------------------------------------------------------------------------------------------------- |
| **Titulo** | Texto | Título que se asignará al registro de la llamada en HubSpot. El agente puede personalizarlo en cada llamada.            |
| **Nota**   | Texto | Nota descriptiva asociada a la llamada. Permite al agente agregar contexto o detalles relevantes sobre la conversación. |

### Campos Personalizados

El administrador puede crear campos adicionales para capturar información específica de su operación. Para agregar un nuevo campo, haga clic en el botón **"+ Agregar campo"** en la parte inferior de la sección.

Al crear un campo personalizado, debe configurar las siguientes propiedades:

| Propiedad             | Opciones                    | Descripción                                                                                                                                       |
| --------------------- | --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Nombre**            | Texto libre                 | El nombre del campo tal como lo verá el agente. Ejemplo: "Resultado de la llamada"                                                                |
| **Descripción**       | Texto libre                 | Una descripción que ayude al agente a entender qué información debe ingresar. Ejemplo: "Describe brevemente el resultado obtenido en la llamada"  |
| **Tipo de dato**      | Texto / Número / Booleano   | Define qué tipo de valor acepta el campo: **Texto** para respuestas libres, **Número** para valores numéricos, **Booleano** para respuestas Sí/No |
| **Valor por defecto** | Texto libre (opcional)      | Un valor inicial que tendrá el campo. El agente puede modificarlo si lo desea                                                                     |
| **Origen del campo**  | Personalizado / Predefinido | **Personalizado**: campo creado manualmente por el administrador. **Predefinido**: campo que ya existe en HubSpot y se mapea directamente         |
| **Campo obligatorio** | Casilla de verificación     | Si se activa, el agente deberá completar este campo obligatoriamente antes de poder enviar el registro                                            |

> **Nota:** Los campos personalizados pueden ser editados o eliminados en cualquier momento desde el panel de administración.

## 3. Registro de Llamadas

Esta sección controla qué tipos de llamadas se registran en HubSpot y permite activar el registro automático.

### Tipos de Llamada

Seleccione para qué tipos de llamada desea habilitar el registro en HubSpot. Los tipos disponibles son:

* **Llamadas entrantes**: Llamadas recibidas por los agentes
* **Llamadas salientes**: Llamadas realizadas por los agentes
* **Llamadas transferidas**: Llamadas que fueron transferidas entre agentes

> **Advertencia:** Si desactiva un tipo de llamada, los agentes no podrán registrar ese tipo de llamada en HubSpot, ni de forma manual ni automática.

### PBX (Business Phone)

El registro en modo PBX se controla mediante un toggle de activación:

* **Toggle ON**: Los agentes del modo PBX podrán registrar llamadas en HubSpot
* **Toggle OFF**: No se habilitará el registro de llamadas PBX en HubSpot

#### Registro Automático PBX

Dentro de la sección PBX, existe un toggle adicional de **Registro automático**:

* **Activado**: Al iniciar una llamada PBX, el sistema creará automáticamente un **engagement** en HubSpot y mostrará la URL del registro al agente. No se requiere intervención del agente.
* **Desactivado**: El registro será manual. Al finalizar la llamada, se mostrará un formulario para que el agente complete los campos antes de enviar el registro.

> **Advertencia:** Cuando el registro automático está activo, se creará un registro en HubSpot por cada llamada que coincida con los tipos seleccionados (entrante, saliente, transferida). Asegúrese de que esto sea el comportamiento deseado antes de activarlo.

### Contact Center

En el modo Contact Center, el registro se configura **por campaña**. El administrador debe seleccionar en cuáles campañas los agentes podrán registrar llamadas en HubSpot.

Cada campaña tiene su propio toggle de activación:

> **Nota:** En HubSpot, las campañas del Contact Center no tienen registro automático habilitado por defecto. El registro será manual (formulario al finalizar la llamada) a menos que se active el registro automático para cada campaña.

Para activar el registro en una campaña específica, simplemente active el toggle correspondiente. Los agentes asignados a esa campaña podrán entonces registrar sus llamadas en HubSpot.

La integración con Freshdesk permite crear **tickets** automáticamente en Freshdesk cuando se registran llamadas desde el WebPhone. A diferencia de HubSpot y Salesforce, Freshdesk requiere la configuración de **credenciales directas** (Access Token y URL de la instancia) y crea tickets de soporte en lugar de registros de llamada o tareas.

## 1. Credenciales de la Integración

A diferencia de HubSpot y Salesforce que utilizan una modalidad de operación, Freshdesk requiere que el administrador configure las credenciales de acceso a la API de Freshdesk.

Para configurar las credenciales, acceda a **Panel de Administración → Integraciones → Freshdesk**.

| Campo                        | Descripción                                                                                                                                                                                          | Ejemplo                            |

| **Access Token**             | Token de autenticación de la API de Freshdesk. Se obtiene desde la configuración de su cuenta de Freshdesk en **Perfil → API Key**.                                                                  | •••••••••••••••••••                |

| ---------- | ----- | ----------- | ----------------- | ---------- |

Freshdesk incluye un campo adicional que viene preconfigurado pero que **puede ser eliminado** por el administrador:

| -------- | ----- | ----------- | ----------------- | ---------- |

El campo **type** define el tipo de ticket que se creará en Freshdesk. El valor por defecto es "Incidentes", pero puede ser modificado según las necesidades de su operación.

> **Nota:** A diferencia de los campos predeterminados (Titulo y Nota) que solo tienen la opción de editar, el campo **type** y cualquier campo personalizado adicional muestran también un botón de eliminar (icono de papelera).

### Campos Personalizados

Para agregar campos adicionales, haga clic en **"+ Agregar campo"** y configure las mismas propiedades que en HubSpot y Salesforce:

* **Nombre**: Nombre visible del campo
* **Descripción**: Texto de ayuda
* **Tipo de dato**: Texto, Número o Booleano
* **Valor por defecto**: Valor inicial (opcional)

Los campos personalizados creados también pueden ser eliminados usando el botón de papelera.

El registro PBX no viene **activado por defecto** con registro automático:

* **Toggle PBX**: Activado por defecto

# Try & Buy (VickyIA)

Este onboarding permite que un cliente se registre y use **20 minutos gratuitos** para probar un **agente de llamadas IA (VickyIA)**.&#x20;

El **control de consumo y bloqueo** lo realiza el **sistema de Billing**, que impide iniciar nuevas llamadas cuando el saldo llega a cero.

<figure><img src="https://content.gitbook.com/content/eN7PEbma2qR1S3UsZOiw/blobs/gOcM3xEWopRBlvMdemmh/image.png" alt=""><figcaption></figcaption></figure>

### Objetivo

* Activar un periodo de prueba con **20 min free**.
* Permitir el uso del agente IA mientras exista saldo de prueba.
* Bloquear nuevas llamadas al agotarse el saldo.
* Redirigir al cliente a la **plataforma principal**, donde podrá **recargar** para continuar usando VickyIA.

#### Componentes Involucrados

**Front Try & Buy (`vicky-onboarding`)**

* **Responsabilidades**:
  * Registro del cliente (formulario de datos personales)
  * Verificación de email mediante código
  * Selección de caso de uso/intereses
  * Configuración del agente virtual
  * Prueba del agente
  * Consulta y visualización del saldo disponible
  * Redirección a plataforma principal cuando el trial se agota

**Billing**

* **Responsabilidades**:
  * Asignación de minutos free (20 minutos iniciales)
  * Validación de cliente/DIDs
  * Medición y consumo de minutos
  * Bloqueo por saldo agotado
  * Gestión de recargas
  * Limpieza de cuentas trial (mediante cron)

**Orquestador de Llamadas**

* **Responsabilidades**:
  * Procesa llamadas según reglas de negocio (tarifas, rutas, planes)
  * Consulta e impacta consumo vía Billing
  * Autoriza o rechaza llamadas según disponibilidad de saldo

**Trunks V2**

* **Responsabilidades**:
  * Capa de enrutamiento/gestión de troncales
  * Escalabilidad y eficiencia en el enrutamiento de llamadas

### Estados del cliente (alto nivel)

| Estado            | Descripción                                    | Fuente de verdad |
| ----------------- | ---------------------------------------------- | ---------------- |
| Trial activo      | Tiene 20 min free disponibles                  | Billing          |
| Trial consumiendo | Está realizando llamadas y descontando minutos | Billing          |
| Trial agotado     | No permite nuevas llamadas                     | Billing          |
| Pago activo       | Tiene saldo por recarga y puede llamar         | Billing          |

## Uso de minutos y reglas de consumo

### Cómo se descuenta el tiempo

El saldo y la autorización para llamar se determinan en el modulo **Billing**.

1. El cliente inicia una llamada desde VickyIA.
2. El **Orquestador de llamadas** aplica reglas de negocio:
   * plan
   * tarifas
   * rutas
3. El Orquestador consulta a Billing para **validar saldo**.

* Verifica que el cliente tenga crédito suficiente
* Calcula el costo estimado de la llamada

4. Si hay saldo:

* Se autoriza la llamada
* La llamada se procesa normalmente
* Billing registra y descuenta consumo según duración/reglas de cobro

5. Si no hay saldo:

* Billing rechaza la autorización
* El Orquestador no permite iniciar la llamada
* El cliente recibe un mensaje de error

* Se impide iniciar nuevas llamadas desde VickyIA

### Comportamiento del sistema

* Billing responde “sin saldo” a la validación previa de llamada.
* El Orquestador no debe permitir crear nuevas ejecuciones de llamada bajo trial agotado.
* El Front debe reflejar el estado y guiar a la recarga.

### Orquestador (Core de telefonia y billing)&#x20;

* No permite crear nuevas ejecuciones de llamada bajo trial agotado
* Retorna error al cliente indicando saldo insuficiente

### Comportamiento esperado en el Front

1. Consultar saldo/estado en Billing.
2. Si saldo = 0 (o Trial agotado):
   * Mostrar “Tus 20 minutos de prueba han finalizado”.

# 🤖 Agentes

Este es el "cerebro" de tus operaciones en Vicky. Aquí podrás crear, configurar y administrar a tus asistentes virtuales (agentes). Desde este módulo defines cómo se llaman, cómo se comportan, qué dicen y qué conocimientos deben tener para atender a tus clientes.

### Funcionalidades Principales

* Gestión Centralizada: Visualiza todos tus agentes en una sola lista con su estado actual y fecha de creación.
* Filtros de Búsqueda: Encuentra rápidamente agentes específicos por nombre, estado o rango de fechas.
* Personalización Avanzada: Define la personalidad (Prompt), el modelo de Inteligencia Artificial (LLM) y el idioma de cada agente.
* Base de Conocimiento: Carga archivos para que tu agente tenga información específica de tu negocio.

#### ¿Cómo usarlo? Guía Paso a Paso

#### 1. Visualizar y Buscar Agentes

<figure><img src="https://2310468671-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FeN7PEbma2qR1S3UsZOiw%2Fuploads%2Fek8vwbA9qHwyNlYwCxQ5%2Fimage.png?alt=media&#x26;token=ca9263d9-7736-4e26-8d9f-7416c47de493" alt=""><figcaption></figcaption></figure>

* Utiliza la barra "Buscar por nombre" para localizar uno rápidamente.
* Usa los filtros de fecha (`Fecha de inicio` y `Fecha de fin`) para ver agentes creados en un periodo específico.
* El botón "Limpiar filtros" restablece la búsqueda

#### 2. Crear un Nuevo Agente

Para añadir un nuevo asistente, haz clic en el botón morado "➕ Nuevo agente" en la esquina superior derecha. Esto abrirá el formulario de configuración.

<figure><img src="https://2310468671-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FeN7PEbma2qR1S3UsZOiw%2Fuploads%2F4QhnIgwhtixWhNbMUWPe%2Fimage.png?alt=media&#x26;token=5d8d419b-bec4-49c2-af0c-baecc0a185d7" alt=""><figcaption></figcaption></figure>

**A. Pestaña "General"**

Aquí configurarás la identidad y el comportamiento base del agente. Completa los siguientes campos:

Información Básica

* Nombre del agente: El nombre interno para identificarlo.
* Categoría: Etiqueta para clasificar al agente (ej. Ventas, Soporte).
* Duración máxima de la llamada: Tiempo límite en segundos para una conversación (por defecto 600s).
* Tiempo de espera para turno: Cuántos segundos esperará el agente una respuesta del usuario antes de hablar de nuevo (por defecto 7s).
* Disposición: Define la rapidez de respuesta (ej. "Rápido").
* Límite diario de llamadas: Tope de llamadas que este agente puede gestionar al día.
* Idioma: El idioma en el que hablará y entenderá el agente.
* Permitir llamadas entrantes: Marca esta casilla si deseas que este agente pueda recibir llamadas, no solo hacerlas.

Configuración del Agente

* Modelo LLM: Selecciona el motor de inteligencia artificial (ej. GPT-5.1).
* Zona Horaria: Configura la hora local del agente (ej. Bogotá, Lima, Quito).

Personalidad y Mensajes

* Primer mensaje: Es el saludo inicial que dará el agente al contestar. Puedes usar variables como `{{nombre_variable}}` para personalizarlo.
* Prompt del agente: Aquí defines "quién es" el agente. Describe su rol, tono y reglas de comportamiento. *Ejemplo: "Eres un asistente útil para atención al cliente..."*.
* Base de conocimiento: Sube un archivo (PDF, TXT, etc.) con información de tu empresa para que el agente la use al responder.

### Tips y Buenas Prácticas

* Uso de Variables: En el *Primer mensaje* y el *Prompt*, usa llaves `{{ }}` para insertar datos dinámicos del cliente (como su nombre).
* Prompt Claro: Cuanto más detallado sea tu *Prompt*, mejor se comportará el agente. Dale ejemplos de qué hacer y qué no hacer.
* Categorías: Usa las categorías para mantener organizados tus agentes si planeas tener muchos (ej. "Campaña Navidad", "Cobranza").

**B. Pestaña "Voz"**

En esta sección definirás cómo suena tu agente. Vicky utiliza tecnología TTS (*Text-to-Speech*) para convertir el texto en audio natural.

* Voz: Despliega la lista para seleccionar el tono y acento que mejor se adapte a tu marca (ej. voces masculinas, femeninas, acentos neutros o locales).
* Velocidad: Ajusta qué tan rápido habla el agente. El rango va de 0.7 (más lento) a 1.2 (más rápido). El valor 1 es la velocidad estándar.
* Probar voz: Haz clic en este botón para escuchar un ejemplo con la configuración actual antes de guardar.

<figure><img src="https://2310468671-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FeN7PEbma2qR1S3UsZOiw%2Fuploads%2FVxaRPS7etTVKwtZghb32%2Fimage.png?alt=media&#x26;token=2428c1cb-e1ff-4bf8-9307-09eb854cfb5d" alt=""><figcaption></figcaption></figure>

**C. Pestaña "Variables de entrada"**

Las variables de entrada sirven para personalizar la conversación antes de que empiece. Permiten inyectar datos externos (como el nombre del cliente o su saldo actual) dentro del guion del agente.

* Nombre: Escribe el nombre técnico de la variable (ej. `nombre_cliente`).
* Descripción: Explica qué dato se espera recibir (ej. "Nombre del usuario que contesta").
* Agregar variable: Usa el botón morado para añadir tantas variables como necesites.

> Nota: Estas variables se pueden usar luego en el *Primer mensaje* o en el *Prompt* usando llaves, por ejemplo: `Hola {{nombre_cliente}}`.

**D. Pestaña "Variables de salida"**

Esta pestaña sirve para recolectar información durante la llamada. Si el agente le pregunta algo al cliente (ej. "¿Está interesado?"), la respuesta se guardará en estas variables para que puedas usarla después en tus reportes o integraciones.

* Haz clic en "Agregar variable de salida" para definir qué datos quieres capturar (ej. `confirmacion_cita`, `motivo_rechazo`).

<figure><img src="https://2310468671-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FeN7PEbma2qR1S3UsZOiw%2Fuploads%2FOx8a1T5Ckb5AtPlDlqaU%2Fimage.png?alt=media&#x26;token=f4055714-958b-42d2-90a6-23d48d08370b" alt=""><figcaption></figcaption></figure>

**E. Pestaña "Herramientas"**

Aquí otorgas "superpoderes" a tu agente para que realice acciones más allá de solo hablar. Se dividen en dos grupos:

1\. Herramientas del Agente (Interacción externa)

* Transferencia de Llamadas: Permite al agente desviar la llamada a un número telefónico humano (ej. un call center) bajo ciertas condiciones.
* Transferir a un agente: Permite pasar la llamada a otro agente virtual dentro del mismo sistema Vicky que esté especializado en otro tema.

2\. Herramientas del Sistema (Acciones internas)

* Terminar conversación: Habilita al agente para colgar la llamada automáticamente cuando detecte que la conversación ha finalizado.
* Reproducir tono de teclado: Permite al agente marcar tonos (DTMF) durante la llamada si es necesario interactuar con otro sistema telefónico.
* Detección de buzón de voz: Si el agente detecta que contestó una máquina contestadora, finalizará la llamada para no hablar solo.

> Tip: Usa los interruptores (toggles) a la derecha de cada opción para activar o desactivar estas capacidades.

**F. Pestaña "Flujo de trabajo"**

Esta es la herramienta visual más potente. En lugar de un solo prompt largo, puedes diseñar la conversación como un diagrama de flujo con pasos lógicos.

* Lienzo Visual: Verás un nodo de "Iniciar". Desde ahí, pulsando el botón `+`, puedes añadir nuevos pasos.
* Tipos de Nodos:
  * Subagent: Crea un "mini-agente" con su propio objetivo específico dentro de la llamada (ej. "Etapa de Saludo", "Etapa de Cierre").
  * Agent transfer / Phone number transfer: Nodos para transferir la llamada.
  * Tool: Ejecutar una herramienta específica.
  * End: Finalizar el flujo.
* Configuración del Nodo: Al hacer clic en un nodo, se abre un panel lateral donde puedes definir su Objetivo, Voz específica para ese paso o su Disposición.

# 📼 Grabaciones

El módulo de Grabaciones es tu centro de auditoría. Aquí se almacena el historial completo de todas las interacciones que han tenido tus agentes. Su objetivo principal es permitirte escuchar, leer y analizar las conversaciones pasadas para evaluar la calidad de la atención, entender mejor a tus clientes o depurar el comportamiento de tus agentes.

### Funcionalidades Principales

* Historial Completo: Acceso a un listado detallado de llamadas entrantes y salientes.
* Filtros Avanzados: Herramientas para encontrar llamadas específicas por fecha, agente o número de teléfono.
* Reproductor de Audio: Escucha la grabación de la llamada directamente desde el navegador.
* Transcripción Automática: Visualiza la conversación en formato de texto (chat) para leer rápidamente lo que se dijo sin tener que escuchar todo el audio.
* Metadatos de Llamada: Información técnica como duración, fechas, estado y números involucrados.

### ¿Cómo usarlo? Guía Paso a Paso

#### 1. Buscar y Filtrar Llamadas

Al ingresar, verás la tabla general de resultados. Si tienes muchas llamadas, utiliza la barra superior para filtrar:

* Agente: Selecciona un asistente virtual específico para ver solo sus gestiones.
* Tipo de llamada: Filtra por llamadas entrantes o salientes ("Saliente", "Entrante", etc.).
* Rango de Fechas: Usa los calendarios "Fecha de inicio" y "Fecha de fin" para acotar la búsqueda.
* Buscar por teléfono: Escribe el número del cliente si buscas una conversación en particular.
* Botón Buscar: Haz clic en el botón azul para aplicar los filtros.

**A. Resumen de la llamada**

# 📅 Campañas

El módulo de Campañas te permite realizar llamadas salientes masivas de forma automatizada. En lugar de marcar uno por uno, Vicky toma una lista de contactos, gestiona las llamadas y conecta a tu Agente de IA cuando el cliente contesta. Es la herramienta ideal para ventas, cobranza, recordatorios o encuestas a gran escala.

### Funcionalidades Principales

* Carga Masiva: Importa miles de contactos mediante archivos Excel o CSV.
* Programación Flexible: Decide si lanzar la campaña de inmediato o agendarla para una fecha futura.
* Gestión de Horarios: Define ventanas de tiempo estrictas para evitar llamar en horas inoportunas.
* Tablero de Control: Monitorea el progreso (dispersión) y el estado de cada campaña en tiempo real.

### ¿Cómo usarlo? Guía Paso a Paso

#### 1. Panel de Control

Al entrar al módulo, verás el listado de todas tus campañas.

* Información clave: Revisa rápidamente el `Asistente` asignado, el `Número DID` de salida y el porcentaje de `Dispersión` (progreso).
* Filtros: Usa la barra superior para buscar campañas por estado, fecha o nombre.

<figure><img src="https://2310468671-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FeN7PEbma2qR1S3UsZOiw%2Fuploads%2FcduSUhSBj0FY5PqXW3To%2Fimage.png?alt=media&#x26;token=7b9f2bfd-dc69-41d4-ad39-01d0ae7d315d" alt=""><figcaption></figcaption></figure>

#### 2. Crear una Nueva Campaña

Haz clic en el botón morado "➕ Nueva campaña" y completa los siguientes pasos:

**A. Datos y Archivo**

* Nombre: Asigna un título para identificar la campaña internamente.
* Archivo: Sube tu base de datos (Excel/CSV).
  * Nota: Asegúrate de descargar la *plantilla de ejemplo*. La columna `phone_number` es obligatoria. Puedes añadir columnas extra (ej. `nombre`, `deuda`) para usarlas como variables en el guion del agente.

**B. Configuración Operativa**

* Asistente IA: Selecciona el agente que atenderá las llamadas.
* Número: Elige la línea telefónica (DID) desde la cual saldrán las llamadas.
* Zona Horaria: Configura la zona horaria correcta (ej. América/Bogotá) para sincronizar los horarios de tus clientes.

**C. Modalidad de Envío**

Aquí defines cuándo iniciará la campaña. Tienes dos opciones en el campo "Modalidad":

1. Inmediata: La campaña intentará arrancar tan pronto la guardes (siempre que esté dentro del horario laboral permitido).
2. Agendada: Te permite programar el lanzamiento para el futuro. Al seleccionar esta opción, aparecerán nuevos campos:
   * Fecha de inicio: Selecciona el día específico en el calendario.
   * Horario de inicio: Define la hora exacta a la que debe arrancar el marcador.

**D. Horario de Trabajo (Opciones Avanzadas)**

Despliega la sección al final para establecer las reglas de marcación. Esto asegura que no se llame fuera de horas laborales, incluso si la campaña está activa.

* Marca los días de la semana permitidos (ej. Lunes a Viernes).
* Establece la hora `Desde` y `Hasta` para cada día.

# 📞 Números telefónicos

Este módulo es el puente entre el mundo telefónico real y tus agentes de IA. Aquí administras tu inventario de números telefónicos (DIDs) y decides qué agente virtual es "dueño" de cada línea. Su función principal es enrutar las llamadas: definir quién contesta cuando un cliente marca a tu empresa y qué número aparece en el identificador de llamadas cuando tus agentes llaman hacia afuera.

### Funcionalidades Principales

* Asignación de Rutas: Vincula un número de teléfono específico a un Agente de IA concreto.
* Control de Llamadas Entrantes: Configura qué "personalidad" (Agente) responderá automáticamente cuando suene el teléfono.
* Permisos de Salida: Autoriza qué agentes tienen permiso para usar ciertos números como identificador de llamada (Caller ID) en sus campañas.
* Gestión de Inventario: Visualiza y busca rápidamente entre todos los números que tienes contratados.

### ¿Cómo usarlo? Guía Paso a Paso

#### 1. Panel de Asignaciones

Al ingresar, verás el listado de tus números configurados.

* Buscador: Utiliza la barra "Buscar por número o agente" para encontrar una configuración específica.
* Filtros: Puedes filtrar por estado para ver solo los números activos.

#### 2. Crear una Nueva Asignación

Para conectar un número con un agente, haz clic en el botón morado "➕ Nueva Asignación". Se abrirá una ventana emergente donde deberás configurar tres puntos clave:

* Número de Teléfono: Selecciona de la lista uno de los números (DIDs) que has adquirido en Vicky.
* Agente Entrante: Elige el Agente de IA que contestará las llamadas que entren a este número.
  * *Ejemplo:* Si asignas el número `+57 300 123 4567` al agente "Soporte Técnico", cada vez que un cliente llame a ese número, hablará con ese agente específico.
* Agentes Salientes: Define qué agentes tienen permiso para usar este número para realizar llamadas al exterior. Esto es útil si tienes varios agentes de ventas y quieres que todos salgan con el mismo número de cabecera de la empresa.

# 📊 Reportes

El módulo de Reportes es tu tablero de control analítico. Aquí es donde los datos se convierten en decisiones. Te permite visualizar el rendimiento de tus agentes de IA en tiempo real o analizar tendencias históricas. Su objetivo es responder preguntas clave como: "¿Están contestando rápido mis agentes?", "¿Qué tan efectivas son mis campañas de ventas?" o "¿Cuánto duran las conversaciones?".

### Funcionalidades Principales

* Visión Dual: Alterna fácilmente entre métricas de llamadas entrantes (Inbound) y salientes (Outbound).
* Selectores de Tiempo: Filtra toda la información por períodos predefinidos (Hoy, Semana, Mes) o define un rango de fechas personalizado.
* KPIs en Tarjetas: Visualiza los indicadores clave de rendimiento (KPIs) en la parte superior con comparativas automáticas respecto al periodo anterior (flechas verdes o moradas).
* Gráficos de Tendencia: Analiza la evolución de tus métricas día a día mediante gráficos visuales.

### ¿Cómo usarlo? Guía Paso a Paso

#### 1. Definir el Periodo de Análisis

En la esquina superior derecha, selecciona el rango de tiempo que deseas auditar:

* Botones Rápidos: Haz clic en "Hoy", "Semana" o "Mes" para una vista rápida.
* Personalizado: Úsalo para elegir fechas específicas (ej. durante una campaña especial).

#### 2. Analizar Reportes Inbound (Llamadas Entrantes)

Haz clic en la pestaña "🎧 Reportes Inbound" para ver cómo tus agentes atienden a los clientes que te llaman.

* Llamadas Atendidas: Volumen total de llamadas que los agentes procesaron.
* ASA (Average Speed of Answer): Velocidad promedio de respuesta. Indica cuánto tarda el agente en empezar a hablar.
* AHT (Average Handle Time): Tiempo promedio de gestión. Cuánto dura la conversación completa.
* Nivel de Servicio: Porcentaje de llamadas contestadas dentro de un umbral aceptable (ej. 20 segundos).
* % Handoff: Porcentaje de llamadas que el agente de IA tuvo que transferir a un humano.

#### 3. Analizar Reportes Outbound (Campañas Salientes)

Haz clic en la pestaña "📞 Reportes Outbound" para medir la efectividad de tus campañas de ventas o cobranza.

* Contactabilidad: De todas las llamadas que hizo Vicky, ¿qué porcentaje contestaron los clientes?
* Conversiones: Número de llamadas que cumplieron el objetivo (ej. "Venta realizada", "Cita agendada").
* Tasa de Éxito: Porcentaje de efectividad sobre los contactos reales.
* Duración Media: Tiempo promedio que los clientes se quedan hablando con el agente.

# 💳 Facturación

El módulo de Facturación es tu centro de control financiero dentro de Vicky. Aquí gestionas el "combustible" de tus agentes: los minutos. Su diseño te permite tener autonomía total para recargar saldo cuando lo necesites, monitorear tu consumo en tiempo real y configurar alertas para evitar que tu operación se detenga por falta de fondos.

### Funcionalidades Principales

* Monitoreo de Saldo: Visualiza al instante cuántos minutos disponibles tienes y su equivalente en dinero (USD).
* Alertas de Saldo Bajo: Configura notificaciones automáticas al correo electrónico cuando tus fondos bajen de cierto límite.
* Recargas Flexibles: Compra minutos a tu medida, ya sea mediante paquetes predefinidos o ingresando una cantidad exacta manual.
* Pasarela de Pagos Integrada: Gestiona tus pagos de forma segura y autónoma directamente desde el portal.
* Historial de Movimientos: Acceso rápido a tus últimas transacciones y recargas.

### ¿Cómo usarlo? Guía Paso a Paso

#### 1. Revisar tu Saldo

En la parte superior de la pantalla verás el panel principal:

* Minutos Disponibles: La cantidad de tiempo de aire que tus agentes pueden usar.
* Balance: El saldo monetario actual en tu cuenta (USD).

#### 2. Configurar Alertas de Seguridad

Para evitar quedarte sin servicio inesperadamente:

1. Ubica la tarjeta "Alertas de Balance".
2. Ingresa un valor en dólares en el campo de texto (ej. `10`).
3. El sistema te enviará automáticamente un correo cuando tu saldo caiga por debajo de esa cantidad (ej. "Te enviaremos un correo... cuando el balance caiga por debajo de 10 USD").

#### 3. Realizar una Recarga

Tienes dos formas de agregar saldo a tu cuenta:

Opción A: Paquetes Sugeridos (Recomendado)

* En la sección derecha, selecciona uno de los paquetes predefinidos (ej. 200 min, 300 min).
* Estos paquetes están diseñados para facilitar la compra rápida.

Opción B: Recarga a la Medida

* Ve a la sección inferior "Recargar tu Cuenta".
* Escribe exactamente cuántos minutos necesitas en el campo "Número de minutos".
* El sistema calculará automáticamente el Total a pagar basándose en el precio por minuto actual (ej. $0.020 USD/min).

Finalizar el Pago: Una vez decidido el monto, haz clic en el botón morado "Continuar al pago". Esto te redirigirá a la pasarela de pagos segura donde podrás completar la transacción con tu método preferido.

#### 4. Consultar Transacciones

Al final de la pantalla encontrarás la sección "Transacciones Recientes".

* Aquí verás los últimos movimientos de tu cuenta.
* Si necesitas un reporte completo, haz clic en el botón "Ver todas".

# ⚙️ Configuración

El módulo de Configuración es el centro de integración de Vicky. Si necesitas que Vicky "hable" con otros sistemas externos (como tu CRM, tu base de datos propia, Zapier o herramientas de análisis), este es el lugar. Aquí generas las llaves de acceso para programadores y configuras notificaciones automáticas hacia otros servidores.

### Funcionalidades Principales

Este módulo se divide en tres pestañas clave:

* Webhooks: Configura notificaciones automáticas para que Vicky envíe datos a tu servidor cada vez que ocurre un evento (ej. al finalizar una llamada).
* Logs: Un historial técnico para monitorear si las notificaciones (webhooks) se enviaron correctamente o si hubo errores.
* API Keys: Genera credenciales de seguridad para permitir que software externo se conecte y controle tu cuenta de Vicky.


# Gestión de Webhooks

Los Webhooks son como "avisos" que Vicky le da a tus sistemas.

* Crear un Webhook:

     * URL: La dirección web de tu servidor donde Vicky enviará la información (ej. `https://tu-empresa.com/api/recibir-llamada`).
     * Descripción: Un nombre para identificar qué hace esta conexión (ej. "Enviar datos a CRM de Ventas").

# Endpoint de Llamadas Salientes

URL producción:  https\://vicky-api.wekall.co

**Endpoint**: `POST /v1/call/outbound-call`\
**Autenticación**: Bearer Token requerido\
**Content-Type**: `application/json`

#### Parámetros de Solicitud <a href="#id-4.2-parametros-de-solicitud" id="id-4.2-parametros-de-solicitud"></a>

**Estructura Básica**

```json
{
  "agentId": "696fe81xxxxxx",
  "toNumber": "+57345xxxxx",
  "conversation_initiation_client_data": {
    "dynamic_variables": {
      "fecha_ultima_interaccion": "12/05/2025",
      "nombre_agente":"vicky",
      "nombre_cliente":"Esneyder",
      "nombre_empresa":"Wekall",
      "numero_cliente":"12345"
    },
    "conversation_config_override": {
      "tts": { "voice_id": "voice_id" },
      "agent": {
        "first_message": "Hola",
        "language": "es",
        "prompt": { "prompt": "Eres un asistente" }
      }
    }
  }
}
```

**Parámetros Obligatorios**

| Campo    | Tipo   | Descripción                                |
| -------- | ------ | ------------------------------------------ |
| agentId  | string | ID único del agente de IA configurado      |
| toNumber | string | Número telefónico destino en formato E.164 |

**Parámetros Opcionales**

| Campo                                                                 | Tipo   | Descripción                                                |
| --------------------------------------------------------------------- | ------ | ---------------------------------------------------------- |
| conversation\_initiation\_client\_data                                | object | Datos adicionales para personalizar la conversación        |
| conversation\_initiation\_client\_data.dynamic\_variables             | object | Variables dinámicas para personalizar el script del agente |
| conversation\_initiation\_client\_data.conversation\_config\_override | object | Configuración personalizada de la conversación             |

**Importante**: El parámetro `conversation_initiation_client_data` es **completamente opcional**. Si no se proporciona, la llamada se realizará con la configuración predeterminada del agente.

Validación de Variables Dinámicas

**Comportamiento del Sistema**

La validación de variables dinámicas **SOLO** se aplica cuando se cumplen **AMBAS** condiciones:

1. Se proporciona `conversation_initiation_client_data` en la solicitud
2. El agente tiene variables dinámicas configuradas

**Escenarios de Validación**

1. **Sin** `conversation_initiation_client_data`:
   * La llamada procede normalmente independientemente de la configuración del agente
   * No se aplica ninguna validación de variables dinámicas
2. **Con** `conversation_initiation_client_data` **pero agente sin variables configuradas**:
   * Las `dynamic_variables` en la solicitud son ignoradas
   * La llamada procede normalmente
3. **Con** `conversation_initiation_client_data` **y agente con variables configuradas**:
   * Las `dynamic_variables` son **obligatorias** en `conversation_initiation_client_data`
   * Las claves deben coincidir **exactamente** con las configuradas en el agente
   * Si faltan variables o no coinciden, se retorna error 400

**Ejemplo de Configuración del Agente**

```json
{
  "conversation_config": {
    "agent": {
      "dynamic_variables": {
        "dynamic_variable_placeholders": {
          "customer_name": "Nombre del cliente",
          "account_number": "Número de cuenta",
          "appointment_date": "Fecha de cita"
        }
      }
    }
  }
}
```

{
  "success": true,
  "call_id": "call-uuid-here",

  "message": "Outbound call initiated successfully"
}

# Remarcación  llamadas

## Políticas de Remarcación

### Descripción General

Las **Políticas de Remarcación** (Redial Policies) permiten configurar estrategias automáticas para reintentar llamadas que no fueron contestadas o fallaron. Esta funcionalidad es esencial para optimizar el alcance a prospectos y mejorar las tasas de conversión en campañas de telemarketing y contact center.

Con las políticas de remarcación, puedes definir reglas específicas que determinen cuándo, cómo y cuántas veces el sistema reintenará una llamada en función de diferentes condiciones (razón de terminación, análisis de IA, estado de llamada, etc.).

***

### Acceso a la Funcionalidad

**Ubicación:** Configuraciones → Políticas de Remarcación

**Ruta en el portal:** `/host/voicebot/settings/me`

<figure><img src="https://2310468671-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FeN7PEbma2qR1S3UsZOiw%2Fuploads%2FNjAbsL8CCzeVlrWAiimJ%2Fimage.png?alt=media&#x26;token=ef7a2d54-8c72-418e-9865-4659d7c29b23" alt=""><figcaption></figcaption></figure>

***

### Crear una Política de Remarcación

#### Paso 1: Iniciar Creación

Desde la pantalla principal de "Políticas de Remarcación", haz clic en el botón **"Crear primera política"** o **"Crear Política"** (si ya existen políticas).

#### Paso 2: Completar Información Básica

**Campo: Nombre**

* **Tipo:** Texto (requerido)
* **Descripción:** Nombre identificador de la política
* **Ejemplo:** "Persistencia para Buzón de Voz", "Estrategia de Agendamiento"
* **Restricciones:** Máximo 255 caracteres, caracteres alfanuméricos y espacios

**Campo: Descripción**

* **Tipo:** Texto (opcional)
* **Descripción:** Explicación detallada del propósito de la política
* **Ejemplo:** "Reintentará automáticamente cuando el cliente no contesta o va al buzón de voz"
* **Restricciones:** Máximo 500 caracteres

**Campo: Política por defecto**

* **Tipo:** Checkbox (opcional)
* **Descripción:** Si está marcado, esta política se aplicará automáticamente a nuevas campañas
* **Comportamiento:** Solo puede existir una política como predeterminada

***

### Configurar Reglas

Las reglas son el corazón de la política. Cada regla define una condición (disparador) y la acción correspondiente que debe ejecutarse.

#### Agregar una Regla

Haz clic en el botón **"+ Agregar Regla"** para añadir una nueva regla a tu política.

**Nota:** Una política debe tener al menos una regla para ser válida.

#### Estructura de una Regla

Cada regla consta de los siguientes campos:

**1. Disparador (Trigger)**

Define la **condición** que activa la regla.

**Tipo de Campo:** Dropdown (requerido)

**Opciones disponibles:**

**a) Razón de Terminación**

Basado en cómo terminó la llamada anterior.

| Valor                        | Descripción                                |
| ---------------------------- | ------------------------------------------ |
| **Buzón de voz (Voicemail)** | La llamada fue al buzón de voz del cliente |
| **No contesta (No Answer)**  | El cliente no respondió la llamada         |
| **Ocupado (Busy)**           | La línea estaba ocupada                    |
| **Fallida (Failed)**         | La llamada falló por problemas técnicos    |
| **Cancelada (Canceled)**     | La llamada fue cancelada                   |

**b) Estado de Llamada**

Basado en el estado actual de la llamada.

| Valor           | Descripción                     |
| --------------- | ------------------------------- |
| **No contesta** | Nadie respondió                 |
| **Ocupado**     | Línea ocupada                   |
| **Fallida**     | Llamada fallida                 |
| **Completada**  | Llamada completada exitosamente |

**c) Análisis IA**

Basado en el análisis inteligente de la IA durante la llamada.

| Valor                                    | Descripción                                  |
| ---------------------------------------- | -------------------------------------------- |
| **Agendar llamada (Callback Scheduled)** | La IA detectó que debe agendarse un callback |
| **Buzón de voz (IA)**                    | La IA identificó un buzón de voz             |
| **Humano contestó**                      | Un humano contestó la llamada                |
| **Interesado**                           | El cliente mostró interés                    |
| **No interesado**                        | El cliente no mostró interés                 |
| **No llamar más (Do Not Call)**          | El cliente solicitó no ser contactado más    |

**d) Valor de Variable**

Basado en variables personalizadas de tu sistema.

***

**2. Valor del Disparador**

Especifica el **valor concreto** de la condición seleccionada.

**Tipo de Campo:** Dropdown o Input de texto (requerido)

**Comportamiento:** El dropdown aparece si el disparador tiene valores predefinidos. Si selecciona "Valor de variable", debe ingresar manualmente el nombre de la variable.

**Ejemplo:**

* Si Disparador = "Razón de Terminación" → Valor = "No contesta"
* Si Disparador = "Análisis IA" → Valor = "Interesado"

***

**3. Acción (Action)**

Define **qué hacer** cuando la regla se cumple.

**Tipo de Campo:** Dropdown (requerido)

**Opciones disponibles:**

**a) Reintentar**

Sistema reintentará la llamada automáticamente.

**Campos adicionales que aparecen:**

* **Máximo de Intentos:** Número de intentos permitidos (1-10)
* **Intervalos:** Minutos de espera entre intentos, separados por coma

**Ejemplo de configuración:**

```
Máximo de Intentos: 3
Intervalos: 15, 60, 1440
```

Esto significa: Reintentar 3 veces con esperas de 15 minutos, 1 hora y 24 horas.

**b) Agendar**

Sistema agendará una llamada futura según una variable de fecha.

**Campos adicionales que aparecen:**

* **Variable de Agenda:** Nombre de la variable que contiene la fecha/hora del callback

**Ejemplo:** `callback_date`

**c) No Reintentar (Stop)**

Sistema no reintentará. La llamada finaliza.

***

#### Campos Específicos por Acción

**Cuando Acción = "Reintentar"**

**Campo: Máximo de Intentos**

* **Tipo:** Número entero
* **Rango:** 1 - 10
* **Descripción:** Cantidad total de intentos permitidos
* **Ejemplo:** 3, 5, 10
* **Valor por defecto:** 3

**Campo: Intervalos (minutos, separados por coma)**

* **Tipo:** Texto
* **Descripción:** Intervalo de tiempo (en minutos) entre cada intento
* **Formato:** Números separados por comas
* **Ejemplos:**
  * `15, 60, 1440` (15 min, 1 hora, 24 horas)
  * `5, 10, 15, 30` (intentos más frecuentes)
  * `60, 240, 1440` (intentos más espaciados)
* **Restricciones:**
  * Números positivos solamente
  * Cantidad de intervalos debe coincidir con máximo de intentos

**Cuando Acción = "Agendar"**

**Campo: Variable de Agenda**

* **Tipo:** Texto (input libre)
* **Descripción:** Nombre de la variable que almacena la fecha/hora del callback
* **Ejemplo:** `callback_date`, `agendado_para`, `fecha_contacto`
* **Restricciones:** Debe existir como variable en el sistema

***

### Eliminar una Regla

En la esquina superior derecha de cada regla, hay un botón **"Eliminar Regla"** de color rojo. Al hacer clic, la regla será removida inmediatamente.

***

### Plantillas de Ejemplo

WeKall ofrece plantillas predefinidas que puedes usar como punto de partida. Estas plantillas contienen configuraciones probadas y mejores prácticas para diferentes escenarios.

#### 1. Persistencia Inteligente (Buzón/No contesta)

**Descripción:** Estrategia para reintentar automáticamente cuando el cliente no contesta o la llamada va al buzón de voz.

**Caso de Uso:** Ideal para prospectos nuevos que aún no conocen tu empresa. Aumenta el alcance sin saturar al cliente.

**Reglas Configuradas:**

| # | Disparador           | Valor        | Acción     | Intentos | Intervalos   |
| - | -------------------- | ------------ | ---------- | -------- | ------------ |
| 1 | Razón de Terminación | Buzón de voz | Reintentar | 3        | 15, 60, 1440 |
| 2 | Razón de Terminación | No contesta  | Reintentar | 2        | 60, 1440     |

**Resumen de Reglas:**

* Si va al buzón: Reintentar en 15 min, 1 hora y 24 horas (máx. 3 veces)
* Si no contesta: Reintentar en 1 hora y 24 horas (máx. 2 veces)

**Cuándo Usar:**

* Primeros contactos con prospectos
* Bases de datos nuevas
* Cuando necesitas maximizar la tasa de alcance

***

#### 2. Agendamiento Automático por IA

**Descripción:** La IA determina cuándo agendar un callback basado en el análisis de la conversación.

**Caso de Uso:** Ideal cuando el cliente está interesado pero no puede hablar en ese momento. La IA automáticamente agenda la próxima llamada.

**Reglas Configuradas:**

| # | Disparador  | Valor           | Acción  | Variable       |
| - | ----------- | --------------- | ------- | -------------- |
| 1 | Análisis IA | Agendar llamada | Agendar | callback\_date |

**Resumen de Reglas:**

* Si la IA detecta que debe agendarse: Agendar automáticamente usando la variable de fecha

**Cuándo Usar:**

* Cuando tienes capacidad de equipo limitada
* Clientes que necesitan follow-up en momentos específicos
* Para mejorar la calidad de las conversaciones

***

#### 3. Stop List / No Molestar

**Descripción:** No reintentar si el cliente solicitó no ser contactado más.

**Caso de Uso:** Cumplimiento legal y respeto de preferencias del cliente. Evita reintentosinnecesarios a clientes desinteresados.

**Reglas Configuradas:**

| # | Disparador  | Valor         | Acción        |
| - | ----------- | ------------- | ------------- |
| 1 | Análisis IA | No llamar más | No Reintentar |
| 2 | Análisis IA | No interesado | No Reintentar |

**Resumen de Reglas:**

* Si el cliente solicitó "No llamar más": Detener reintentos
* Si el cliente está "No interesado": Detener reintentos

**Cuándo Usar:**

* Siempre, para cumplir con regulaciones (GDPR, CCPA, etc.)
* Proteger la reputación de tu marca
* Respetar las preferencias del cliente

***

### Mejores Prácticas

#### 1. Intervalos Realistas

* Comienza con intervalos cortos (15-30 min) para ser ágil
* Aumenta a intervalos largos (horas, días) según avanza
* Evita rebotar demasiado rápido (máximo 5 min)

#### 2. Máximo de Intentos Moderado

* 2-3 intentos es generalmente suficiente
* Más de 5 intentos puede ser contraproducente
* Considera la tolerancia del cliente

#### 3. Usar Plantillas como Base

* Las plantillas están optimizadas por expertos
* Personaliza según tus necesidades específicas
* Prueba y mide resultados

#### 4. Documentar el Propósito

* Usa una descripción clara y detallada
* Facilita la comprensión del propósito a futuro
* Útil para el equipo y auditorías

#### 5. Política por Defecto

* Selecciona una política coherente con tu estrategia general
* Asegúrate que sea apropiada para la mayoría de casos
* Puedes cambiarla en cualquier momento

***

### Preguntas Frecuentes (FAQ)

**¿Puedo tener múltiples políticas?** Sí, puedes crear tantas políticas como necesites. Esto es útil para diferentes tipos de campañas.

**¿Puedo editar una política después de crearla?** Sí, cualquier política puede ser editada en cualquier momento desde la lista de políticas.

**¿Qué pasa si la IA no puede determinar la acción?** Si la IA no encuentra coincidencia en tus reglas, la llamada se tratará según tu configuración predeterminada del sistema.

**¿Puedo cambiar la política por defecto?** Sí, simplemente marca la opción "Política por defecto" en la política que desees establecer como predeterminada.

**¿Los intervalos deben ser exactos?** El sistema intentará respetar los intervalos, pero pueden variar según la disponibilidad del sistema (variación de ±5 minutos).

**¿Hay límite de reglas por política?** Técnicamente no hay límite, pero se recomienda usar 3-5 reglas para mantener la política clara y manejable.

# bots de flujo

Los **Bots de Flujo** ejecutan conversaciones predefinidas mediante una estructura de nodos y transiciones. Son ideales para procesos estructurados donde el camino de la conversación está claramente definido.

## ¿Cómo Funciona un Bot de Flujo?

Un bot de flujo sigue una secuencia de **nodos** conectados por **transiciones**. Cada nodo representa una acción o pregunta, y las transiciones definen el camino que sigue la conversación según la respuesta del usuario.

```
[Inicio] → [Saludo] → [Menú Principal] → [Opción 1] → [Fin]
                                        → [Opción 2] → [Captura Datos] → [Fin]
                                        → [Opción 3] → [Transferir Agente]
```

> **📸 Captura de pantalla:** *Ejemplo de flujo conversacional en el editor*

## Tipos de Nodos

Los nodos disponibles para construir flujos son:

### Nodos Básicos

| Nodo         | Descripción                                            |
| ------------ | ------------------------------------------------------ |
| **Inicio**   | Punto de entrada del flujo. Todo flujo debe tener uno. |
| **Mensaje**  | Envía un mensaje de texto al usuario.                  |
| **Pregunta** | Envía un mensaje y espera la respuesta del usuario.    |

# bots de ia

Los **Bots de IA** utilizan modelos de lenguaje avanzados (GPT-4o, GPT-4o-mini) para mantener conversaciones naturales con los usuarios. Son capaces de entender contexto, extraer información, detectar intenciones y responder de forma inteligente.

***

## ¿Cómo Funciona un Bot de IA?

El bot de IA recibe el mensaje del usuario, lo procesa a través de un pipeline de inteligencia artificial y genera una respuesta contextualizada. El pipeline sigue este orden de prioridad:

1. **Redirecciones** - Detecta palabras clave para acciones específicas
2. **Procesamiento multimedia** - Analiza imágenes y transcribe audios
3. **Extracción de variables** - Captura datos del usuario
4. **Confirmación** - Confirma datos capturados con el usuario
5. **Detección de intenciones** - Identifica la intención del mensaje
6. **Intenciones configuradas** - Usa respuestas predefinidas (ahorra tokens)
7. **Búsqueda en bases de conocimiento** (RAG) - Busca información relevante
8. **Generación de respuesta IA** - Genera respuesta con OpenAI
9. **Ejecución de herramientas** - Ejecuta funciones si la IA lo solicita
10. **Guardado de memoria** - Persiste información importante

***

## Configuración del Bot de IA

### Prompt del Sistema

El **prompt del sistema** es la instrucción principal que define el comportamiento del bot. Aquí defines:

* **Rol** del bot (ej: asistente de ventas, soporte técnico)
* **Personalidad** y tono de comunicación
* **Instrucciones** específicas de comportamiento
* **Restricciones** (qué no debe hacer)
* **Contexto** del negocio

> **📸 Captura de pantalla:** *Editor del prompt del sistema*

### Modelo de IA

Selecciona el modelo de lenguaje a utilizar:

| Modelo          | Descripción                   | Uso Recomendado                                |
| --------------- | ----------------------------- | ---------------------------------------------- |
| **GPT-4o**      | Modelo más avanzado y preciso | Conversaciones complejas, análisis de imágenes |
| **GPT-4o-mini** | Modelo rápido y económico     | Conversaciones simples, alto volumen           |

### Parámetros Avanzados

| Parámetro              | Descripción                               | Rango      |
| ---------------------- | ----------------------------------------- | ---------- |
| **Temperatura**        | Controla la creatividad de las respuestas | 0.0 - 1.0  |
| **Max tokens**         | Límite de tokens en la respuesta          | 100 - 4096 |
| **Umbral de fallback** | Confianza mínima antes de transferir      | 0.0 - 1.0  |

# intenciones y redirecciones

Los bots de IA pueden configurarse con **Intenciones** y **Redirecciones** para controlar su comportamiento de forma precisa. Estas funcionalidades permiten definir respuestas específicas y acciones automáticas basadas en lo que dice el usuario.

***

## Intenciones

Las **intenciones** son respuestas predefinidas que el bot utiliza cuando detecta un tema específico en el mensaje del usuario. Su principal ventaja es que **ahorran tokens de IA** al no necesitar generar una respuesta desde cero.

### Acceder a Intenciones

1. Abre el detalle de un bot de IA.
2. Haz clic en la pestaña **Intenciones**.

> **📸 Captura de pantalla:** *Vista de intenciones del bot*

### Crear una Intención

1. Haz clic en **"+ Nueva Intención"**.
2. Completa el formulario:

| Campo              | Descripción                                                   |
| ------------------ | ------------------------------------------------------------- |
| **Nombre**         | Identificador único de la intención (ej: `horarios_atencion`) |
| **Descripción**    | Descripción de cuándo debe activarse                          |
| **Palabras clave** | Palabras o frases que activan esta intención                  |
| **Respuesta**      | El texto que el bot enviará al usuario                        |
| **Activa**         | Si la intención está habilitada o no                          |

3. Haz clic en **Guardar**.

> **📸 Captura de pantalla:** *Formulario de creación de intención*

### Ejemplo de Intención

| Campo          | Valor                                                                   |
| -------------- | ----------------------------------------------------------------------- |
| Nombre         | `horarios_atencion`                                                     |
| Descripción    | Cuando el usuario pregunta por horarios                                 |
| Palabras clave | horario, horarios, hora de atención, cuándo abren                       |
| Respuesta      | Nuestro horario de atención es de Lunes a Viernes de 8:00 AM a 6:00 PM. |

{% hint style="info" %}
Cuando una intención coincide, el bot usa la respuesta predefinida en lugar de consultar a la IA, lo que reduce significativamente el consumo de tokens.
{% endhint %}

***

## Redirecciones

Las **redirecciones** son acciones automáticas que se disparan cuando el usuario escribe una palabra clave específica. Tienen la **mayor prioridad** en el pipeline de procesamiento del bot.

### Acceder a Redirecciones

1. Abre el detalle de un bot de IA.
2. Haz clic en la pestaña **Redirecciones**.

> **📸 Captura de pantalla:** *Vista de redirecciones del bot*

### Tipos de Redirecciones

| Tipo           | Descripción                                                     |
| -------------- | --------------------------------------------------------------- |
| **Transferir** | Transfiere la conversación a un agente humano o área específica |
| **Cerrar**     | Cierra la conversación del bot                                  |
| **Escalar**    | Escala la conversación con prioridad alta                       |
| **Feedback**   | Inicia un flujo de recolección de feedback                      |

### Crear una Redirección

1. Haz clic en **"+ Nueva Redirección"**.
2. Completa el formulario:

| Campo              | Descripción                                             |
| ------------------ | ------------------------------------------------------- |
| **Palabra clave**  | Palabra o frase que activa la redirección               |
| **Tipo de acción** | Transferir, Cerrar, Escalar o Feedback                  |
| **Mensaje**        | Texto que el bot enviará antes de ejecutar la acción    |
| **Destino**        | Área o agente de destino (solo para transferir/escalar) |
| **Activa**         | Si la redirección está habilitada                       |

3. Haz clic en **Guardar**.

> **📸 Captura de pantalla:** *Formulario de creación de redirección*

### Ejemplo de Redirección

| Campo          | Valor                                                                            |
| -------------- | -------------------------------------------------------------------------------- |
| Palabra clave  | agente, asesor, hablar con alguien                                               |
| Tipo de acción | Transferir                                                                       |
| Mensaje        | Entiendo, te voy a comunicar con uno de nuestros asesores. Un momento por favor. |
| Destino        | Área de soporte                                                                  |

***

## Prioridad de Procesamiento

El orden de prioridad al procesar un mensaje es:

1. **Redirecciones** (máxima prioridad)
2. **Intenciones configuradas**
3. **Generación de respuesta IA** (menor prioridad)

Esto significa que si un mensaje coincide con una redirección, se ejecutará la acción correspondiente sin consultar intenciones ni IA.

{% hint style="warning" %}
Asegúrate de que las palabras clave de las redirecciones sean lo suficientemente específicas para evitar activaciones no deseadas.
{% endhint %}

### Bots de Flujo

* Verificar que el flujo sigue la ruta correcta
* Probar cada opción del menú
* Comprobar la captura de variables
* Verificar las validaciones
* Probar las condiciones y bifurcaciones
* Verificar los mensajes de error

### Bots de IA

* Comprobar que el prompt del sistema funciona correctamente
* Probar la captura de variables
* Verificar las intenciones configuradas

* Evaluar la calidad de las respuestas
* Probar con diferentes idiomas

{% hint style="info" %}

4. **Prueba las redirecciones**: Escribe las palabras clave configuradas para verificar que las acciones se ejecutan.
5. **Prueba la transferencia**: Verifica que el bot indica correctamente cuándo transferirá a un agente.

## Organización por Categorías

Organiza tus mensajes predefinidos en categorías para facilitar su búsqueda:

* **Saludos**: Mensajes de bienvenida y presentación
* **Despedidas**: Mensajes de cierre de conversación
* **Soporte**: Respuestas frecuentes de soporte técnico
* **Ventas**: Información de productos y servicios
* **Seguimiento**: Mensajes de seguimiento post-venta
* **Categorías personalizadas**: Crea las que necesites

{% hint style="info" %}
Los mensajes predefinidos son compartidos entre todos los agentes de tu organización. Cualquier agente puede usarlos en sus conversaciones.
{% endhint %}

## Variables en Mensajes Predefinidos

Puedes incluir variables en los mensajes que se reemplazarán automáticamente:

| Variable       | Se reemplaza por            |
| -------------- | --------------------------- |
| `{{contacto}}` | Nombre del contacto actual  |
| `{{agente}}`   | Nombre del agente que envía |

Ejemplo:

```
Hola {{contacto}}, soy {{agente}}. ¿En qué puedo ayudarte hoy?
```

# Llamadas

## Llamadas de WhatsApp

La funcionalidad de **Llamadas** te permite realizar y recibir llamadas de voz directamente desde WhatsApp dentro de la plataforma Messenger Hub.

***

### ¿Cómo Funcionan las Llamadas?

Cuando un contacto te llama por WhatsApp o tú realizas una llamada, la conversación de voz se integra directamente en tu chat activo, permitiéndote atender sin salir de la plataforma.

### Recibir Llamadas de WhatsApp

#### Llamada Entrante

Cuando un contacto te llama por WhatsApp:

1. Aparece una **notificación de llamada entrante** en tu pantalla.
2. Escuchas un **tono de llamada**.
3. Puedes **Aceptar** o **Rechazar** la llamada.

> **📸 Captura de pantalla:** *Modal de llamada entrante*

#### Durante la Llamada

Una vez aceptada la llamada, puedes:

* **Silenciar tu micrófono** (mute)
* **Ver el tiempo de duración** de la llamada
* **Colgar** cuando termines la conversación
* **Continuar viendo el chat** mientras hablas

> **📸 Captura de pantalla:** *Controles durante la llamada*

***

### Realizar Llamadas a Contactos

Para llamar a un contacto de WhatsApp:

1. Abre el chat del contacto.
2. Haz clic en el botón de **llamada** (📞) en la parte superior del chat.
3. Espera a que el contacto conteste.

   <div data-gb-custom-block data-tag="hint" data-style="info" class="hint hint-info"><p>Solo puedes llamar a contactos que tengan WhatsApp y hayan iniciado una conversación contigo previamente.</p></div>

***

### Permisos del Navegador

La primera vez que uses las llamadas, el navegador te pedirá permiso para acceder a tu **micrófono**:

1. Haz clic en **Permitir** cuando aparezca el mensaje.
2. Si rechazaste el permiso por error, puedes habilitarlo desde la configuración del navegador.

> **📸 Captura de pantalla:** *Solicitud de permiso del micrófono*

***

### Historial de Llamadas

Todas las llamadas realizadas y recibidas quedan registradas en el historial del chat:

* **Llamada entrante** (recibida)
* **Llamada saliente** (realizada)
* **Llamada perdida** (no contestada)
* **Duración** de la llamada

> **📸 Captura de pantalla:** *Historial de llamadas en el chat*

***

### Solución de Problemas

#### No escucho al contacto

1. Verifica que tu **volumen** esté activado.
2. Revisa que los **altavoces** de tu computadora funcionen correctamente.
3. Intenta colgar y volver a llamar.

#### El contacto no me escucha

1. Verifica que tu **micrófono** no esté silenciado (mute).
2. Revisa que el navegador tenga **permiso** para usar el micrófono.
3. Prueba tu micrófono en otra aplicación para verificar que funcione.

#### La llamada se corta o tiene mala calidad

1. Verifica tu **conexión a Internet** (se recomienda conexión estable).
2. Cierra otras pestañas o aplicaciones que consuman mucho ancho de banda.
3. Si usas WiFi, acércate al router o usa conexión por cable.

### Código de Embebido

Existen dos opciones para embeber el widget en tu sitio web:

#### Opción 1: Script Externo (Recomendado)

Carga el script desde el servidor. Es más eficiente y soporta notificaciones en tiempo real con WebSocket.

```html
<!-- Cargar Socket.IO para WebSocket en tiempo real -->
<script src="https://cdn.socket.io/4.7.2/socket.io.min.js"></script>

<!-- Cargar el widget de Live Chat -->
<script src="https://tu-dominio.com/live-chat-widget.js"></script>
<script>
  window.initLiveChatWidget({
    customer: 'tu-cliente',
    apiUrl: 'https://api.tu-dominio.com',
    position: 'bottom-right',
    primaryColor: '#007bff'
  });
</script>
```

#### Opción 2: Código Directo

Incluye todo el código del widget en un solo script. Es más simple de configurar y funciona sin WebSocket (solo HTTP).

**Pasos para usar el código:**

1. Selecciona la pestaña **"Script Externo"** o **"Código Directo"**.
2. Haz clic en el botón **Copiar** (📋).

# Custom tools

La configuración de **Custom Tools** (Herramientas Personalizadas) permite definir funciones externas que los bots de IA pueden invocar durante las conversaciones para consultar APIs, ejecutar acciones o recuperar información en tiempo real.

***

***

## ¿Qué son las Custom Tools?

Las Custom Tools implementan el patrón de **Function Calling** de OpenAI, permitiendo que el bot de IA:

1. Identifique cuándo necesita información externa durante la conversación.
2. Invoque automáticamente la herramienta con los parámetros correctos.
3. Utilice la respuesta de la herramienta para generar una respuesta al usuario.

### Ejemplo de Uso

```
Usuario: "¿Cuál es el estado de mi pedido #12345?"

Bot: [Internamente invoca la herramienta "consultar_pedido" con id=12345]
     [Recibe: { estado: "En camino", fecha_entrega: "2025-03-10" }]

Bot: "Tu pedido #12345 está en camino. La fecha estimada de entrega es el 10 de marzo de 2025."
```

## Solución de Problemas

### No recibo mensajes

1. Verifica que el webhook esté configurado correctamente en Meta.
2. Confirma que la URL sea accesible públicamente.
3. Revisa que el token de verificación coincida.
4. Verifica la suscripción a los eventos `messages`.

### Los mensajes no se envían

1. Verifica que el token de acceso sea válido y no haya expirado.
2. Confirma que el número esté verificado en Meta.
3. Si la ventana de 24h expiró, usa un template aprobado.
4. Revisa que tengas créditos suficientes.

### Los estados no se actualizan

1. Verifica la suscripción al evento `message_status` en Meta.
2. Revisa que el webhook esté procesando correctamente los callbacks.

Messenger Hub es una plataforma de mensajería empresarial multi-canal que permite gestionar conversaciones con clientes a través de WhatsApp, Instagram, Telegram y Live Chat desde una única interfaz.

</details>

Messenger Hub es compatible con las versiones más recientes de:

* Google Chrome (recomendado)
* Mozilla Firefox
* Microsoft Edge
* Safari

</details>

<details>

<summary>¿Puedo usar Messenger Hub en el celular?</summary>

Sí, la plataforma cuenta con un diseño responsivo que se adapta a dispositivos móviles y tablets. Sin embargo, para una experiencia óptima se recomienda usar un navegador de escritorio.

</details>

## Chats

<details>

<summary>¿Qué es la ventana de 24 horas de WhatsApp?</summary>

Es una política de Meta que solo permite enviar mensajes libres dentro de las 24 horas posteriores al último mensaje del cliente. Después de este período, solo se pueden enviar templates aprobados por Meta.

</details>

<details>

<summary>¿Por qué no puedo enviar mensajes a un contacto?</summary>

Posibles razones:

* La ventana de 24 horas expiró (usa un template aprobado).
* El contacto bloqueó tu número.
* Problemas con la conexión de la API de WhatsApp.
* No tienes créditos suficientes.

</details>

<details>

<summary>¿Cómo puedo saber si un mensaje fue leído?</summary>

Los estados de los mensajes se muestran con marcas de verificación:

* ✓ Enviado
* ✓✓ Entregado
* ✓✓ (azul) Leído

</details>

<details>

<summary>¿Puedo transferir un chat a otro agente?</summary>

Sí, desde las opciones del chat (⋮) selecciona **Transferir** y elige el agente o área de destino.

</details>

<details>

<summary>¿Qué pasa cuando cierro un chat?</summary>

Al cerrar un chat debes seleccionar una tipificación (categoría de cierre). El chat pasará al estado "cerrado" y se podrá consultar en el historial. Si el contacto envía un nuevo mensaje, se creará un nuevo chat.

</details>

El contexto de la conversación (variables capturadas, resumen) se preserva y se muestra al agente como un banner en la parte superior del chat.

No, los mensajes enviados en modo de prueba no se guardan en la base de datos ni afectan las métricas del bot.

# Solución de problemas

Guía para resolver los problemas más comunes en Messenger Hub.

***

## Problemas de Conexión

<details>

<summary>No puedo iniciar sesión</summary>

**Síntomas**: La página de login no carga o muestra un error de autenticación.

**Soluciones**:

1. Verifica que tus credenciales sean correctas (usuario y contraseña).
2. Asegúrate de que tu cuenta no esté bloqueada (contacta al administrador).
3. Limpia la caché y cookies del navegador.
4. Intenta en una ventana de incógnito.
5. Verifica tu conexión a Internet.

> **📸 Captura de pantalla:** *Error de autenticación común*

</details>

<details>

<summary>La plataforma se muestra lenta o no carga</summary>

**Soluciones**:

1. Verifica tu conexión a Internet.
2. Cierra pestañas innecesarias del navegador.
3. Limpia la caché del navegador.
4. Intenta con otro navegador compatible.
5. Contacta al administrador si el problema persiste.

</details>

<details>

<summary>Se desconecta frecuentemente (WebSocket)</summary>

**Síntomas**: Aparece un indicador de desconexión o los mensajes no llegan en tiempo real.

**Soluciones**:

1. Verifica la estabilidad de tu conexión a Internet.
2. Si usas una VPN, intenta sin ella.
3. Revisa si hay un firewall bloqueando las conexiones WebSocket.
4. Recarga la página.

</details>

***

## Problemas con Mensajes

<details>

<summary>Los mensajes no se envían</summary>

**Síntomas**: El mensaje queda en estado "enviando" o muestra un error.

**Soluciones**:

1. Verifica que la conexión WebSocket esté activa (indicador verde).
2. Para WhatsApp: Verifica que la ventana de 24h no haya expirado.
3. Verifica que el número del contacto sea válido.
4. Revisa que tengas créditos suficientes.
5. Si envías multimedia, verifica que el archivo no supere el tamaño máximo.

</details>

<details>

<summary>No recibo mensajes de WhatsApp</summary>

**Síntomas**: Los mensajes enviados por clientes no aparecen en la plataforma.

**Soluciones**:

1. Verifica que el webhook esté configurado correctamente en Meta.
2. Confirma que la URL del webhook sea accesible públicamente.
3. Revisa que el token de verificación coincida.
4. Verifica la suscripción a los eventos `messages` en Meta.
5. Contacta al administrador para revisar los logs del servidor.

</details>

<details>

<summary>Los estados de mensajes no se actualizan</summary>

**Síntomas**: Los mensajes quedan siempre en "enviado" sin cambiar a "entregado" o "leído".

**Soluciones**:

1. Verifica la suscripción al evento `message_status` en Meta.
2. Asegúrate de que el contacto no tenga desactivadas las confirmaciones de lectura.
3. Contacta al administrador para revisar los webhooks.

</details>

<details>

<summary>Los archivos multimedia no se envían</summary>

**Soluciones**:

1. Verifica que el formato del archivo sea compatible con el canal.
2. Revisa que el tamaño no supere el límite permitido.
3. Para audios: El sistema convierte automáticamente a formato OGG para WhatsApp.
4. Verifica la conexión con el almacenamiento S3.

</details>

***

## Problemas con Bots

<details>

<summary>El bot no responde</summary>

**Síntomas**: El bot está activo pero no genera respuestas.

**Soluciones**:

1. Verifica que el bot esté en estado **Activo**.
2. Confirma que no haya conflictos con otro bot en el mismo canal y número.
3. Para bots de IA: Verifica que haya créditos disponibles.
4. Revisa la configuración del bot (prompt del sistema, modelo, etc.).
5. Prueba el bot usando la función **Probar Bot**.

</details>

<details>

<summary>El bot de IA da respuestas incorrectas</summary>

**Soluciones**:

1. Revisa y ajusta el **prompt del sistema** con instrucciones más claras.
2. Ajusta la **temperatura** (menor = más preciso, mayor = más creativo).
3. Agrega o actualiza las **bases de conocimiento** con información relevante.
4. Configura **intenciones** con respuestas predefinidas para temas frecuentes.
5. Verifica que las **redirecciones** no estén capturando mensajes incorrectamente.

</details>

<details>

<summary>El bot de flujo se queda atascado</summary>

**Soluciones**:

1. Abre el **Editor de Flujos** y verifica que todos los nodos estén conectados.
2. Revisa el indicador de **validación** del flujo.
3. Verifica que las condiciones estén correctamente configuradas.
4. Asegúrate de que haya nodos de **Fin** o **Transferir** al final de cada ruta.
5. Prueba el flujo completo usando el panel de pruebas.

</details>

<details>

<summary>Las variables no se capturan correctamente</summary>

**Soluciones**:

1. Verifica la configuración de las variables de captura (nombre, tipo, validación).
2. Para bots de IA: Revisa que la descripción de cada variable sea clara.
3. Para bots de flujo: Verifica los nodos de captura de variable y sus validaciones.

</details>

# Configuración notificaciones

## Sistema de Notificaciones

<figure><img src="https://2310468671-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FeN7PEbma2qR1S3UsZOiw%2Fuploads%2FP6LPPOoSxjZGA70Dz688%2Fimage.png?alt=media&#x26;token=09e964c7-1704-4340-ae51-b1cd0ef01784" alt=""><figcaption></figcaption></figure>

### Descripción General

El sistema de notificaciones te mantiene informado sobre eventos importantes dentro de la plataforma, como cuando el saldo de una cuenta es bajo o cuando ocurre un error en algún servicio.

Las alertas llegan de dos formas:

* **Al instante en la aplicación:** aparecen mientras tienes la app abierta, sin necesidad de recargar.
* **Por correo electrónico:** si lo configuras, recibirás un correo aunque no estés usando la app en ese momento.

Como administrador, también puedes decidir qué tipo de eventos generan notificaciones, a quién llegan por correo y si cada alerta está activa o pausada.

***

### Dónde Encontrarlo

#### Centro de notificaciones (campana)

En la **barra superior derecha** de la aplicación verás una campana 🔔.

* Si tienes notificaciones nuevas sin leer, la campana mostrará un número rojo.
* Haz clic en la campana para abrir el panel de notificaciones.

#### Pantalla de configuración

1. En el menú de la izquierda, busca la sección **IAM**.
2. Haz clic en **Notificaciones**.
3. Se abrirá la pantalla donde puedes crear y gestionar las reglas de notificación.

***

### Cómo Usar el Centro de Notificaciones

#### Ver mis notificaciones

1. Haz clic en la campana en la esquina superior derecha.
2. Se abre un panel con todas tus notificaciones recientes.
3. Las más nuevas aparecen primero en la lista.
4. Un punto rojo junto a una notificación indica que todavía no la has leído.
5. Haz clic en cualquier notificación de la lista para ver sus detalles completos en la parte inferior del panel.

#### Marcar una notificación como leída

* En la lista, cada notificación no leída tiene un botón con una marca de verificación (✓) a la derecha. Haz clic en él para marcarla como leída.
* También puedes abrir el detalle de la notificación y hacer clic en **"Marcar como leída"** desde ahí.

#### Marcar todas como leídas de una vez

* En la parte superior del panel, si tienes notificaciones pendientes, verás el botón **"Marcar todo"**.
* Haz clic en él para marcar todas tus notificaciones como leídas en un solo paso.

#### Cerrar el panel

* Haz clic en la **X** en la esquina superior del panel, o
* Haz clic en cualquier parte de la pantalla fuera del panel, o
* Presiona la tecla **Escape** en tu teclado.

#### Indicador de conexión

En la parte superior del panel verás un pequeño punto de color junto al texto de estado:

| Color    | Significado                                                 |
| -------- | ----------------------------------------------------------- |
| Verde    | La app está conectada y recibiendo alertas al instante      |
| Amarillo | Conectando, espera un momento                               |
| Rojo     | Desconectado o error — las alertas en tiempo real no llegan |
| Gris     | Sin conexión activa                                         |

Si ves el estado en rojo, actualiza la página. La conexión se restablece automáticamente en la mayoría de los casos.

***

### Cómo Configurar las Notificaciones

Esta sección es para **administradores** que desean definir qué alertas se envían y a quién.

#### Paso 1 — Activar los tipos de alerta

Al ingresar a la pantalla de configuración, verás una sección llamada **"Códigos de evento predefinidos"** con los tipos de alerta disponibles:

* **Saldo bajo:** avisa cuando una cuenta o bot no tiene saldo suficiente.
* **Error de sistema:** notifica cuando ocurre un fallo crítico en algún servicio.

Cada uno tiene una casilla de verificación. Márcala para activar ese tipo de alerta, desmárcala para desactivarlo.

#### Paso 2 — Configurar cada regla

Cuando activas un evento, aparece una tarjeta con sus opciones:

**Nombre del evento** Es el identificador de la alerta (por ejemplo, `LOW_BALANCE`). Puedes dejarlo tal como está o escribir uno personalizado si sabes exactamente qué evento quieres capturar.

**Regla habilitada** Una casilla que activa o pausa esa regla. Si la desactivas, el evento no generará ninguna alerta aunque la regla exista.

**Canales de entrega** Elige cómo llegará la notificación:

* **En la app:** la notificación aparece en la campana mientras el usuario tiene la aplicación abierta.
* **Por correo:** se envía un correo electrónico a los destinatarios que definas.

Puedes activar uno o ambos canales al mismo tiempo.

**Destinatarios por correo** Si activaste el canal de correo, escribe aquí las direcciones a las que quieres enviar las alertas. Puedes agregar varias separándolas con comas.

Ejemplo: `admin@empresa.com, soporte@empresa.com`

#### Paso 3 — Agregar más reglas personalizadas

Si necesitas configurar un evento adicional que no está en la lista predefinida:

1. Haz clic en el botón **"Agregar regla"** en la parte inferior.
2. Se creará una tarjeta nueva en blanco.
3. Escribe el nombre del evento, elige los canales y agrega los correos si aplica.

#### Paso 4 — Guardar los cambios

Cuando termines de configurar las reglas, haz clic en **"Guardar cambios"** (botón morado en la esquina superior derecha). Verás un mensaje de confirmación si todo salió bien.

***

### Botones de Acción

| Botón               | Qué hace                                                                       |
| ------------------- | ------------------------------------------------------------------------------ |
| **Guardar cambios** | Guarda toda la configuración actual de notificaciones                          |
| **Probar conexión** | Envía una prueba para verificar que las reglas están funcionando correctamente |
| **Recargar**        | Vuelve a cargar la configuración guardada sin perder lo que tienes en pantalla |
| **Eliminar regla**  | Borra esa regla específica de la configuración                                 |

***

### Información de la Organización

En la parte superior de la pantalla de configuración verás un resumen con:

* **Organización:** el nombre de tu empresa u organización registrada.
* **ID de organización:** un identificador único de tu cuenta (solo informativo, no necesitas cambiarlo).

***

### Casos de Uso

**Ejemplo 1 — Alerta de saldo bajo** El equipo de operaciones quiere saber de inmediato cuando una cuenta queda sin saldo. Activan la alerta "Saldo bajo", eligen el canal "En la app" y "Por correo", y agregan los correos del equipo de finanzas. Así, todos reciben el aviso al instante.

**Ejemplo 2 — Error de sistema solo para el equipo técnico** El área de TI quiere recibir alertas de errores críticos, pero solo ellos. Activan "Error de sistema", eligen solo el canal "Por correo" y agregan únicamente los correos del equipo técnico.

**Ejemplo 3 — Pausar una alerta temporalmente** Durante una migración planificada se esperan errores controlados. El administrador desactiva la casilla "Regla habilitada" del evento de errores para que no lleguen alertas innecesarias durante ese periodo.

***

### Tips y Consejos

* Si activas el canal de correo en una regla, asegúrate de agregar al menos un destinatario antes de guardar, de lo contrario no podrás guardar los cambios.
* Puedes tener varias reglas activas al mismo tiempo para distintos tipos de eventos.
* Si no ves la campana o el panel no abre, actualiza la página.
* Cuando el punto de estado en el panel de notificaciones está en verde, significa que las alertas llegan en tiempo real sin necesidad de recargar.
* Usa el botón **"Probar conexión"** después de guardar tu configuración para confirmar que todo funciona.

***

### Preguntas Frecuentes

**¿Por qué no me llegan notificaciones aunque la regla está activa?** Verifica que la casilla "Regla habilitada" esté marcada. También asegúrate de que el indicador de conexión en la campana esté en verde. Si está en rojo, actualiza la página.

**¿Puedo configurar que las alertas lleguen solo por correo y no en la app?** Sí. En la configuración de cada regla, simplemente desmarca el canal "En la app" y deja marcado solo "Por correo".

**¿Cuántas reglas puedo crear?** No hay un límite definido. Puedes crear tantas reglas como necesites para los distintos tipos de eventos de tu organización.

**¿Qué pasa si ingreso un correo con formato incorrecto?** El sistema te mostrará un aviso antes de guardar indicando cuál correo no tiene el formato correcto. Corrígelo y vuelve a intentar guardar.

**¿Puedo eliminar toda la configuración?** Sí. Al hacer clic en "Eliminar regla" cuando existe una configuración guardada, el sistema te pedirá confirmación. Si aceptas, se eliminará toda la configuración de notificaciones para tu organización.

**¿Si no hay ninguna configuración previa, qué pasa?** La pantalla cargará automáticamente una plantilla inicial con los eventos más comunes ya preconfigurados. Solo tienes que ajustarla según tus necesidades y guardar.

**¿Cuántas notificaciones se guardan en la campana?** Se muestran las notificaciones más recientes. Las más antiguas se van reemplazando cuando llegan nuevas.

***

### Mensajes de Error

**"No tienes permisos para gestionar configuraciones de notificaciones"** Tu usuario no tiene el nivel de acceso necesario para cambiar esta configuración. Contacta a un administrador de la plataforma.

**"Tu sesión no es válida para este módulo"** Tu sesión expiró o no tiene los permisos correctos. Cierra sesión, vuelve a ingresar y prueba de nuevo.

**"Debes configurar al menos una regla para guardar"** No puedes guardar si no tienes ninguna regla activa. Agrega al menos una y luego guarda.

**"La regla \[nombre] debe tener al menos un canal activo"** Cada regla necesita tener activado al menos uno de los canales (en la app o por correo). Márcalo y guarda de nuevo.

**"La regla \[nombre] requiere al menos un correo destinatario"** Si activaste el canal de correo en una regla, debes escribir al menos una dirección de correo válida.

**"El correo \[dirección] no tiene un formato válido"** Revisa que la dirección ingresada tenga el formato correcto: `nombre@dominio.com`.

**"No fue posible cargar la configuración"** Hubo un problema al obtener tu configuración. Haz clic en **"Reintentar"** para intentarlo de nuevo. Si el error persiste, contacta a soporte técnico.
